## Рефакторинг (Лабораторная работа 6)

### Найденные code smells
- Функция `process_checkout` слишком длинная и делает всё сразу.
- Дублирование циклов по items.
- Магические числа и строки (0.21, "SAVE10", 200 и т.д.) прямо в коде.
- Сложная вложенная логика скидок.

### Применённые улучшения
- Вынесены константы (TAX_RATE, DEFAULT_CURRENCY).
- Логика разбита на маленькие функции: валидация, подсчёт суммы, скидки, генерация ID.
- `process_checkout` стала короткой и читается как сценарий сверху вниз.
- Убрано дублирование, применены принципы DRY и KISS.

### Результат
Код стал значительно проще читать и поддерживать. Добавление новых купонов или изменение налога теперь требует правки в одном месте. Все тесты продолжают проходить.
